# 羽球手记

> 只服务羽毛球爱好者的成长记录小程序
> 记录打球、看清水平、学规则防受伤

## 核心功能

- **打球记录**：打完一场，15 秒记一笔（日期/时长/强度/感受/不适部位/场地/搭子）
- **水平自评**：中羽民间等级 1.0-5.0，对照特征自评，约球时对水平用
- **本周目标**：默认每周 3 场，进度条可视化，可自定义
- **智能提醒**：基于记录自动提醒（过量、连续不适、频率下降等）
- **累计数据**：球龄、累计场次、累计时长，纯成就感
- **知识手册**：基础规则、技术要领、防伤指南，随时查阅

## 信息架构（4 个 tab）

```
tabBar
├─ 首页 index      等级徽章 + 本周进度 + 智能提醒 + 最近3场
├─ 记录 record     记一笔弹窗 + 本周进度 + 历史记录列表
├─ 知识 guide      规则 / 技术 / 防伤 三大板块（可折叠）
└─ 我的 profile    累计数据 + 等级自评 + 每周目标 + 个人档案
```

## 数据结构

### `user_profile`

```js
{
  openid, nick_name, avatar_url,
  gender, age, height_cm, weight_kg,
  badminton_level: 2.0,    // 中羽民间等级 1.0-5.0
  play_since: "2024-03",   // 球龄起点
  weekly_goal: 3,          // 每周目标场次
  level_updated_at,
  created_at, updated_at,
}
```

### `manual_record`（打球记录）

```js
{
  user_id,
  record_date,
  duration_min,        // 时长（分钟）
  intensity,           // 1摸鱼 2正常 3猛冲
  feeling,             // 1良好 2有点累 3不适
  sore_parts: [],      // 不适部位：膝盖/肩膀/脚踝/手肘/腰部/足底
  venue: "",           // 场地（可选）
  partners: [],        // 搭子（可选）
  note: "",            // 备注（可选）
  created_at,
}
```

## 中羽民间等级（自评标准）

| 等级 | 名称 | 特征 |
|------|------|------|
| 1.0 | 入门 | 频次少，能打过网 |
| 1.5 | 初学 | 知道让对方接不到，苍蝇拍握法 |
| 2.0 | 业余初级 | 经常打，懂规则 |
| 3.0 | 业余中级 | 能跑全场但不流畅 |
| 4.0 | 业余中高 | 有双打意识，反手能过渡 |
| 5.0 | 业余高手 | 技术无短板，能组织战术 |

> 参考中羽在线业余分级，非官方认证。官方等级需到中国羽协指定机构付费测试。

## 智能提醒规则

| 触发条件 | 提醒 |
|---------|------|
| 同一部位连续 2 次不适 | 建议休息 3-5 天，不缓解就医 |
| 本周猛冲 ≥ 3 场 | 强度偏高，建议休息 |
| 本周超出目标 2 场以上 | 注意膝盖肩膀恢复 |
| 近 2 周频率明显下降 | 别松懈，约一场 |
| 默认 | 本周还差 X 场达标 |

## 快速开始

### 1. 开通云开发

1. 用微信开发者工具打开本项目
2. 替换 `project.config.json` 中的 `appid`
3. 替换 `app.js` 中的云环境 ID
4. 开通云开发，创建环境

### 2. 创建数据库集合

| 集合名 | 权限 | 说明 |
|--------|------|------|
| `user_profile` | 仅创建者可读写 | 用户档案 |
| `manual_record` | 仅创建者可读写 | 打球记录 |

### 3. 部署云函数

右键 `cloudfunctions/` 下每个文件夹 → 「上传并部署：云端安装依赖」：
- `login`
- `record`
- `dashboard`
- `reportList`

### 4. 运行

点击「编译」预览。

## 设计理念

- **垂直**：只做羽毛球，不做通用运动
- **极简记录**：15 秒记一笔，预填默认值
- **数据有价值**：智能提醒把记录翻译成行动建议
- **知识兜底**：即使不记录，知识手册也有查阅价值

## 提审注意
